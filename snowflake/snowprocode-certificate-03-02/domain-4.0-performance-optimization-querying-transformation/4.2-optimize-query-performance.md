# ⚡ Snowflake Query Performance Optimization – Key Features

---

## 🚀 Query Acceleration Service (QAS)
- Improves performance of **long-running queries**
- Uses **additional compute resources (serverless)** to speed up query execution  

### ✔️ Best For:
- Large scans  
- Complex queries with heavy filtering  

### ✔️ Benefits:
- Faster query response time  
- No manual tuning required  
- Automatically scales  

---

## 🔍 Search Optimization Service
- Improves performance of **selective queries (point lookups)**  
- Creates a **search access path** for faster data retrieval  

### ✔️ Best For:
- Queries with:
  - `WHERE column = value`
  - Highly selective filters  
- Semi-structured data (JSON)

### ✔️ Benefits:
- Reduces scan time  
- Speeds up lookup queries significantly  

---

## 🧱 Clustering Keys
- Defines how data is **physically organized** in micro-partitions  

### ✔️ Purpose:
- Improve **partition pruning**  
- Reduce data scanned during queries  

### ✔️ Example:
```sql
CREATE TABLE sales (
  id INT,
  region STRING,
  sale_date DATE
)
CLUSTER BY (region, sale_date);

```

## 🧱 Clustering Keys – When to Use
- Large tables  
- Frequent filtering on specific columns  

---

## 📊 Materialized Views
- Precomputed and stored query results  
- Automatically maintained by Snowflake  

### ✔️ Benefits:
- Faster query performance  
- Avoid repeated computation  

### ✔️ Example:
```sql
CREATE MATERIALIZED VIEW mv_sales_summary AS
SELECT region, SUM(amount) AS total_sales
FROM sales
GROUP BY region;
```
