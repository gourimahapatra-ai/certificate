# ⚡ Snowflake Caching – Quick Notes

---

## 🧠 Use Snowflake Caching
Snowflake uses multiple layers of caching to **improve query performance** and **reduce compute cost**.

---

## 📌 Query Result Cache
- Stores **final query results**
- If the same query is executed again (with no data changes), Snowflake returns results instantly  

### ✔️ Key Points:
- No compute cost (no warehouse needed)  
- Valid for **24 hours**  
- Query must be **identical**  
- Underlying data must not change  

---

## 📌 Metadata Cache
- Stores **table and micro-partition metadata**  
- Used for **query optimization and pruning**  

### ✔️ Benefits:
- Faster query planning  
- Efficient partition pruning  
- Reduces data scanning  

---

## 📌 Warehouse Cache (Local Disk Cache)
- Stores **data blocks on the virtual warehouse nodes**  
- Also called **data cache**  

### ✔️ Key Points:
- Used when the same data is queried again  
- Faster than reading from cloud storage  
- Cache is cleared when:
  - Warehouse is suspended  
  - Warehouse is resized  

---

## 🔄 How It Works Together

1. **Query Result Cache** → Returns full result instantly  
2. **Warehouse Cache** → Reuses previously scanned data  
3. **Metadata Cache** → Optimizes query execution plan  

---

## 🧠 Pro Tips
- Use consistent queries to benefit from **result cache**  
- Avoid unnecessary warehouse suspension if reusing data frequently  
- Combine caching with proper query design for best performance
