# ⚙️ Snowflake Automated Data Ingestion – Quick Notes

---

## 🔄 Perform Automated Data Ingestion
Automates **continuous data loading and processing** with minimal manual intervention.

---

## 🚀 Snowpipe
- Serverless, auto-ingest service  
- Loads data **continuously** as files arrive in a stage  
- Triggered via:
  - Event notifications (S3, Azure, GCS)  
  - REST API  

### ✔️ Key Features:
- Near real-time ingestion  
- No warehouse required (uses Snowflake-managed compute)  
- Pay-per-use (credits consumed)

---

## ⚡ Snowpipe Streaming
- Enables **low-latency row-level ingestion** (no files needed)  
- Directly streams data into Snowflake tables  

### ✔️ Benefits:
- Faster than file-based ingestion  
- Ideal for real-time applications (IoT, logs, events)  
- Reduces dependency on staging files  

---

## 🌊 Streams
- Tracks **data changes (CDC – Change Data Capture)** in tables  
- Captures:
  - Inserts  
  - Updates  
  - Deletes  

### ✔️ Use Cases:
- Incremental data processing  
- Building pipelines (ELT)  
- Auditing changes  

---

## ⏱️ Tasks
- Used to **schedule SQL statements**  
- Can run:
  - On a schedule (cron)  
  - After another task (dependency chain)  

### ✔️ Example Use:
- Run transformation every hour  
- Process new data from streams  

---

## 🔄 Dynamic Tables
- Automatically **maintain and refresh derived tables**  
- Similar to materialized views but more flexible  

### ✔️ Features:
- Incremental refresh  
- Declarative pipeline definition  
- Reduces manual ETL effort  

---

## 🌐 Openflow *(Not in Exam Scope Yet)*
- Framework for building **data pipelines and integrations**  
- Still evolving → not tested until generally available (GA)  

---

## 🔗 How They Work Together

1. **Snowpipe / Snowpipe Streaming** → Ingest data  
2. **Streams** → Track changes  
3. **Tasks** → Trigger processing  
4. **Dynamic Tables** → Maintain transformed data  

---

## 🧠 Pro Tips
- Use **Snowpipe** for file-based near real-time ingestion  
- Use **Snowpipe Streaming** for real-time event ingestion  
- Combine **Streams + Tasks** for incremental pipelines  
- Prefer **Dynamic Tables** for simplified pipeline management
