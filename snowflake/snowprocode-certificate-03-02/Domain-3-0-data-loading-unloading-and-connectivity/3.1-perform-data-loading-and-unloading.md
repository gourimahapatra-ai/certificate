# 📥 Snowflake Data Loading & Unloading – Quick Notes

---

## 🔄 Perform Data Loading and Unloading
- **Loading (Ingress):** Bringing data into Snowflake tables  
- **Unloading (Egress):** Exporting data from Snowflake to external storage  
- Primary command used: `COPY INTO`

---

## 📄 File Formats
- Define how data files are interpreted during load/unload  
- Supported formats:
  - CSV  
  - JSON  
  - Avro  
  - Parquet  
  - ORC  
  - XML  

### ✔️ Key Options:
- Field delimiter  
- File compression (gzip, bz2, etc.)  
- Header handling  
- Null value handling  

```sql
CREATE FILE FORMAT my_csv_format
TYPE = 'CSV'
FIELD_DELIMITER = ','
SKIP_HEADER = 1
NULL_IF = ('NULL', 'null')
COMPRESSION = 'GZIP';

CREATE FILE FORMAT my_json_format
TYPE = 'JSON'
STRIP_OUTER_ARRAY = TRUE
COMPRESSION = 'AUTO';

CREATE FILE FORMAT my_parquet_format
TYPE = 'PARQUET';

CREATE FILE FORMAT my_avro_format
TYPE = 'AVRO';

CREATE FILE FORMAT my_xml_format
TYPE = 'XML';

COPY INTO my_table
FROM @my_stage
FILE_FORMAT = (FORMAT_NAME = my_csv_format);

```

## 📦 Create and Use Stages

Stages are **temporary storage locations** for data files before loading or after unloading.

---

### 🏠 Internal Stages
- Managed by Snowflake  
- Types:
  - User stage (`@~`)  
  - Table stage (`@%table_name`)  
  - Named internal stage  
- No external cloud setup required  

---

### ☁️ External Stages
- Stored in cloud storage:
  - Amazon S3  
  - Azure Blob Storage  
  - Google Cloud Storage  
- Require:
  - Credentials (IAM roles, keys)  
  - Storage integration  

---

### 🔐 Server-Side Encryption
- Automatically encrypts data in external storage  
- Managed by cloud provider or Snowflake  
- Ensures data security at rest  

---

### 📂 Directory Tables
- Metadata layer on stages  
- Stores file-level details:
  - File name  
  - Size  
  - Last modified timestamp  
- Useful for tracking and automation  

---

## 📥 COPY INTO Command

### ✔️ Loading Data into Tables:
```sql
COPY INTO target_table
FROM @stage_name
FILE_FORMAT = (TYPE = 'CSV');

```

**Pro Tip**
- Use Parquet/ORC for large-scale analytics (better performance & cost)
- Use JSON/Avro for semi-structured data
- Use CSV for simple, flat data exchange
