# 🔐 Snowflake Data Security & Governance – Quick Notes

---

## 🎭 Data Masking (Column-Level Security)
- Protects sensitive data by **masking column values**
- Uses **Dynamic Data Masking policies**
- Different users see different data based on roles  
- Example: Show full SSN to admins, partial to analysts  

---

## 🧍 Row-Level Security
- Controls **which rows a user can see**
- Implemented using **Row Access Policies**
- Filters data dynamically based on user role/context  
- Example: Sales users see only their region’s data  

---

## 📊 Column-Level Security
- Restricts access to specific columns
- Implemented using:
  - Masking policies  
  - Secure views  
- Prevents exposure of sensitive fields (PII, financial data)

---

## 🏷️ Object Tagging
- Assigns **metadata tags** to objects (tables, columns, etc.)
- Used for:
  - Data classification (PII, Sensitive, Confidential)  
  - Governance and auditing  
- Tags can drive masking and access policies  

---

## 🔒 Privacy Policies
- Enforce **data protection and compliance rules**
- Work with tags and masking policies  
- Help meet regulations like GDPR, HIPAA  

---

## 🛡️ Trust Center
- Centralized view of **security, compliance, and governance posture**
- Provides insights into:
  - Data protection  
  - Access controls  
  - Compliance status  

---

## 🔑 Encryption Key Management
- Data is encrypted:
  - **At rest (AES-256)**
  - **In transit (TLS)**
- Supports:
  - Snowflake-managed keys  
  - Customer-managed keys (Tri-Secret Secure – Business Critical)  

---

## 🚨 Alerts
- Triggered based on **conditions or thresholds**
- Example: Notify when query cost exceeds limit  
- Helps in proactive monitoring  

---

## 🔔 Notifications
- Sends messages via:
  - Email  
  - External integrations (e.g., cloud messaging services)  
- Often used with alerts or tasks  

---

## 🔁 Data Replication and Failover
- Replicates data across **regions/accounts**
- Supports:
  - Disaster recovery  
  - Business continuity  
- Enables **failover/failback** in outages  

---

## 🔍 Data Lineage
- Tracks **data flow and transformations**
- Shows:
  - Source → transformations → target  
- Helps with:
  - Impact analysis  
  - Debugging  
  - Compliance  

---
