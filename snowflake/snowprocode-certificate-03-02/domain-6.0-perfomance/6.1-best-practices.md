# Snowflake Best Practices (Cost & Performance Optimization)

Snowflake follows a **pay-as-you-use model**, so cost optimization focuses on:
- **Compute (Virtual Warehouses)**
- **Storage (Tables, Time Travel, Fail-safe)**

---

## 1. Warehouse (Compute) Optimization

###  Right Sizing
- Start with **SMALL / MEDIUM** warehouses
- Scale **up (size)** for heavy queries
- Scale **out (multi-cluster)** for high concurrency

> ✔ Complex queries → Scale **Up**  
> ✔ Many users → Scale **Out**

---

###  Auto Suspend / Resume
- Enable:
  - `AUTO_SUSPEND = 60–300 sec`
  - `AUTO_RESUME = TRUE`

>  Prevents charges for idle warehouses

---

### Multi-Cluster Warehouses
- Best for **concurrent workloads**
- Configure:
  - `MIN_CLUSTER = 1`
  - `MAX_CLUSTER = N`

>  Don’t oversize warehouse when concurrency is the issue

---

###  Query Optimization
- Avoid `SELECT *`
- Use **filters (WHERE clause) early**
- Use **result caching**
- Avoid repeated execution of same queries

---

##  2. Storage Optimization (Tables)

###  Data Types
- Use efficient data types:
  - `NUMBER` instead of `VARCHAR` for numeric values
  - Define proper precision

>  Smaller data = lower storage + faster scans

---

###  Reduce Data Duplication
- Avoid unnecessary table copies
- Use:
  - **Zero-copy cloning**
  - **Views instead of duplicate tables**

---

###  Clustering
- Use only for **large tables**
- Improves **partition pruning**

>  Clustering adds compute cost

---

###  File Format Optimization
- Use compressed formats:
  - **Parquet**
  - **ORC**

> ✔ Reduces storage and improves performance

---

##  3. Monitoring & Cost Control

###  Key Views
- `ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY`
- `ACCOUNT_USAGE.STORAGE_USAGE`
- `QUERY_HISTORY`

---

###  Best Practices
- Build **cost monitoring dashboards**
- Track:
  - Idle warehouses
  - Expensive queries
- Use **Resource Monitors**:
  - Set credit limits
  - Trigger alerts or auto-suspend

---

##  4. Time Travel & Retention

###  Time Travel
- Access historical data (1–90 days)

---

###  Cost Impact
- Old data is retained → increases storage cost

---

###  Best Practices
- Set retention based on need:
  - Dev/Test → **0–1 day**
  - Production → **1–7 days**

---

###  Fail-safe
- Additional **7 days retention (non-configurable)**
- Used for disaster recovery

>  Still contributes to storage cost

---

##  Bonus Best Practices

###  Caching
- Result cache → no compute cost
- Metadata cache → faster performance

---

###  Use Temporary / Transient Tables
- No fail-safe → reduces storage cost

---

###  Data Loading
- Use `COPY INTO` instead of `INSERT` for bulk loads

---

##  Key Takeaway

Optimize:
- ✅ Compute (Warehouses)
- ✅ Storage (Tables & Formats)
- ✅ Monitoring (Usage Tracking)
- ✅ Retention (Time Travel)

>  Goal: **Minimize cost while maximizing performance in Snowflake**
